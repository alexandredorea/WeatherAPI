# Frequently Asked Questions - FAQ 


**Pergunta:** Porque você optou por fazer uma arquitetura mais complexa e robusta para algo tão simples? Era apenas um CRUD utilizando uma validação por intermédio de uma API.

**Resposta:** Obviamente que uma arquitetura anemica atenderia - até mais facilmente ao que foi pedido - mas optei por não fazer uma arquitetura BOLOVO para dar uma ideia do conhecimento que tenho. O que me levou a incluir mockups de telas, detalhamento mais completo para desenvolvedores novos que estão estudando sobre os assuntos que foram abordados neste projetos, etc.


-----


**Pergunta:** Como eu faço o download do código fonte e configuro meu computador para rodar este projeto?

**Resposta:** Primeiramente você vai precisar ter instalado em seu computador o [Visual Studio 2017 Community][IDEVS2017] e o [SQL Server 2017 Express][IDESQL2017] em sua máquina local, conforme descrevi no quadro de [Recursos Utilizados][RepositoryWeatherAPI].

Depois que as IDEs estiverem devidamente instaladas e configuradas, você deve acessar o repositório com o fonte [WeatherAPI][RepositoryWeatherAPI] e baixá-los, ou se preferir, pode também fazer um *Fork* deste repositório para o seu Github.

![Pergunta_02][FAQ_02]

Não há dificuldades para execução da aplicação - ainda que se utilize como banco o SQL Server - pois para fins de avaliação, optou-se em realizar um attachment do banco de dados (arquivo *mdf) para não criar uma nova instância local em sua máquina.
